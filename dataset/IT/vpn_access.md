# TechNova Solutions Pvt. Ltd.

# VPN and Remote Access Policy

**Document Version:** 1.0
**Effective Date:** January 1, 2026
**Department:** Information Technology & Information Security
**Applicable To:** All Employees, Contractors, Consultants, and Third-Party Users

---

# 1. Purpose

The purpose of this VPN and Remote Access Policy is to establish secure procedures and requirements for accessing TechNova Solutions Pvt. Ltd. resources remotely. This policy ensures that remote access to company networks, systems, applications, and data remains secure and compliant with organizational security standards.

Remote access introduces additional security risks, and therefore all users must adhere to the requirements defined in this policy.

---

# 2. Scope

This policy applies to:

* Full-time employees
* Contract employees
* Interns
* Consultants
* Third-party vendors
* Remote workers
* Hybrid employees

The policy covers access to:

* Corporate networks
* Cloud applications
* Internal servers
* Development environments
* Databases
* Administrative systems
* Communication platforms

---

# 3. Definitions

## VPN (Virtual Private Network)

A VPN creates an encrypted connection between a user's device and company infrastructure, ensuring secure transmission of data.

## Remote Access

Remote access refers to connecting to company systems from outside the corporate network.

## Multi-Factor Authentication (MFA)

A security mechanism requiring multiple methods of authentication before granting access.

---

# 4. Eligibility for Remote Access

Remote access privileges may be granted to:

* Employees approved by their managers.
* Employees working remotely.
* Business travelers.
* Support personnel.
* Authorized vendors.

Access approval depends on business necessity and security requirements.

---

# 5. Remote Access Approval Process

The remote access process consists of the following steps:

### Step 1

Employee submits a remote access request.

### Step 2

Manager reviews business justification.

### Step 3

Information Security reviews risk assessment.

### Step 4

IT department provisions access.

### Step 5

User receives credentials and instructions.

---

# 6. VPN Requirements

All remote connections to company resources must use the approved corporate VPN.

Employees must:

* Install approved VPN software.
* Use company-issued credentials.
* Enable multi-factor authentication.
* Maintain updated security software.
* Connect only through authorized VPN gateways.

---

# 7. Authentication Requirements

All remote users must utilize:

* Username and password
* Multi-factor authentication
* Device verification
* Security certificates where applicable

Authentication credentials must never be shared.

---

# 8. Password Requirements

VPN passwords must:

* Contain at least 12 characters.
* Include uppercase letters.
* Include lowercase letters.
* Include numbers.
* Include special characters.

Passwords expire every 90 days.

Passwords must not:

* Be written down.
* Be shared.
* Be reused.
* Contain personal information.

---

# 9. Company Device Requirements

Employees accessing company resources remotely must use:

* Company-issued laptops
* Company-approved mobile devices
* Managed workstations

Devices must have:

* Antivirus protection
* Firewall enabled
* Full-disk encryption
* Automatic updates
* Endpoint protection software

---

# 10. Personal Device Usage

Personal devices may only be used if:

* Approved by IT.
* Enrolled in device management systems.
* Encrypted.
* Protected by antivirus software.
* Compliant with company security standards.

The company reserves the right to revoke access to personal devices at any time.

---

# 11. Public Network Usage

Employees must avoid using:

* Public Wi-Fi networks
* Shared computers
* Internet cafés
* Untrusted networks

If public internet access is unavoidable:

* VPN usage is mandatory.
* Sensitive activities should be minimized.
* Additional security precautions should be taken.

---

# 12. Remote Work Security Requirements

Employees working remotely must:

* Maintain a secure workspace.
* Lock devices when unattended.
* Prevent unauthorized access.
* Use secure internet connections.
* Protect confidential information.

Family members and unauthorized individuals may not use company devices.

---

# 13. Session Management

Remote sessions are automatically managed according to security standards.

| Parameter                | Configuration |
| ------------------------ | ------------- |
| Maximum Session Duration | 12 Hours      |
| Idle Timeout             | 15 Minutes    |
| Failed Login Attempts    | 5 Attempts    |
| Password Expiration      | 90 Days       |

---

# 14. Access Restrictions

Remote users are prohibited from:

* Sharing accounts.
* Installing unauthorized software.
* Disabling security controls.
* Downloading confidential data unnecessarily.
* Accessing systems beyond their authorization level.

Violations may result in access suspension.

---

# 15. Monitoring and Logging

The company monitors remote access activities including:

* Login attempts
* Session duration
* Accessed resources
* Geographic locations
* Failed authentication attempts
* Security alerts

Monitoring is conducted to protect company assets and comply with regulatory requirements.

---

# 16. Security Incident Reporting

Employees must immediately report:

* Lost devices
* Credential compromise
* Unauthorized access
* Suspicious activities
* Malware infections
* Phishing attacks

Reports should be submitted to:

* Information Security Team
* IT Helpdesk
* Immediate Manager

---

| Service                 | Availability Target |
| ----------------------- | ------------------- |
| VPN Infrastructure      | 99.9%               |
| Authentication Services | 99.9%               |
| Remote Support          | Business Hours      |

# 18. Third-Party Remote Access

Third-party vendors requiring remote access must:

* Sign confidentiality agreements.
* Receive formal approval.
* Use approved devices.
* Follow company security policies.
* Accept activity monitoring.

Third-party access is granted only for the required duration.

---

# 19. Access Suspension and Revocation

Remote access privileges may be suspended due to:

* Security violations
* Policy violations
* Employment termination
* Extended inactivity
* Business changes

Access revocation occurs immediately upon employee separation.

---

# 20. Compliance Requirements

Employees must comply with:

* Company policies
* Security procedures
* Data protection regulations
* Confidentiality agreements
* Industry standards

Failure to comply may result in disciplinary action.

# 22. Frequently Asked Questions

### Q1. Is VPN access mandatory for remote work?

Yes. All remote access to company resources must occur through the approved VPN.

### Q2. Can employees use personal laptops?

Only with prior approval and compliance with security requirements.

### Q3. What happens if I forget my VPN password?

Employees should contact the IT Helpdesk.

### Q4. Can VPN credentials be shared with colleagues?

No. Credential sharing is strictly prohibited.

### Q5. What should I do if my company laptop is stolen?

Immediately report the incident to the Information Security Team and IT Helpdesk.
